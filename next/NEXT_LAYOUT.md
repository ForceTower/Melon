**Role:** Act as a Senior Principal Staff Engineer. Your task is to bootstrap a polyglot monorepo for a project named **Melon**. 

**Context:**
Melon is a full-stack application being rebuilt from scratch. It consists of a TypeScript backend, native Android/iOS apps powered by Kotlin Multiplatform (KMP), and potential future web frontends. 

**Tech Stack:**
* **Backend:** TypeScript + Bun.
* **Mobile Shared Logic:** Kotlin Multiplatform (KMP).
* **Android App:** Native Kotlin + Jetpack Compose.
* **iOS App:** Native Swift + SwiftUI.
* **Infrastructure:** Docker / Docker Compose.

**Monorepo Tooling Boundaries (CRITICAL):**
Do not mix the build tools. 
1.  The root of the repository is a **Bun workspace**. It manages the `apps/api` and TS `packages/`.
2.  The root also contains a **Gradle project**. The `settings.gradle.kts` manages `apps/android` and `packages/shared-kmp`. Gradle should NOT touch the TypeScript code. Bun should NOT touch the Kotlin code.

**Step 1: Scaffold the Root Structure**
Create the following directory tree and the base configuration files:

```text
melon/
├── apps/
│   ├── android/          
│   ├── ios/              
│   └── api/              
├── packages/
│   ├── shared-kmp/       
│   └── shared-types/     
├── infra/
│   └── docker/           
├── package.json          (Bun workspace config pointing to apps/* and packages/*)
├── tsconfig.json         (Base TS config)
├── build.gradle.kts      (Root Gradle config for Android/KMP plugins)
├── settings.gradle.kts   (Includes :apps:android and :packages:shared-kmp modules)
└── gradle.properties     (Enable KMP, AndroidX, and configure JVM arguments)
```

**Step 2: Scaffold the Bun/TypeScript Backend (`apps/api`)**
* Initialize a basic Bun server in `apps/api/src/index.ts`.
* Ensure its `package.json` relies on the workspace.
* Create a basic `infra/docker/docker-compose.yml` that can spin up this Bun API and a local PostgreSQL database.

**Step 3: Scaffold the KMP Architecture (`packages/shared-kmp`)**
Implement a Feature-Driven KMP Architecture with an Umbrella module. Create the following structure inside `packages/shared-kmp`:

```text
packages/shared-kmp/
├── build-logic/            
│   └── src/main/kotlin/melon.kmp-library.gradle.kts (Convention plugin for submodules)
├── core/                   
│   ├── network/            (Ktor setup)
│   └── database/           (Room or SQLDelight setup)
├── features/               
│   ├── auth/               (Depends on core:network)
│   └── dashboard/          
└── umbrella/               
    └── build.gradle.kts    (Depends on all features/core and exports a single XCFramework for iOS)
```
* Ensure the `build-logic` convention plugin is used by `core` and `features` modules to avoid Gradle boilerplate.
* The `umbrella` module must be the ONLY module that generates an iOS framework.

**Step 4: Scaffold the Native Mobile Apps**
* **Android (`apps/android`):** Create a basic Android application module (Jetpack Compose). It must depend on project(`:packages:shared-kmp:umbrella`) or specific features.
* **iOS (`apps/ios`):** Create a base Xcode project (SwiftUI). Configure it to consume the `SharedKmp.xcframework` generated by the `packages/shared-kmp:umbrella` module.
